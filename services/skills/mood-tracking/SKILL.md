# NPC 情绪追踪流程

NPC 每次对玩家回复后，都需要上报本轮的情绪变化。此步骤由 `update_npc_mood` 工具完成。

## 调用时机

- 在生成阶段最后一轮（tool_choice 允许 `update_npc_mood` 的轮次），调用一次 `update_npc_mood`，随后连同最终文本一起结束。
- 若本轮是纯工具查询阶段（尚未给玩家最终文本），**不要**调用 update_npc_mood。

## 参数

```json
{
  "emotion": "<从可用情绪中选，可为空串>",
  "title": "<从可用称号中选，可为空串>",
  "favorability_delta": -100 ~ 100,
  "reason": "<1~2 句中文，说明选定情绪/好感度变化的理由>"
}
```

### emotion

- 从 prompt 中 `可用情绪` 段落的合法值中挑选。
- 可以为空字符串 `""` 表示沿用上一轮情绪。
- 选最贴合本轮对话氛围的一个情绪，不要机械地固定在"中性"。

### title

- 从 `可用称号` 中选，可为空（通常保留不变）。
- 只有关系等级变化时才切换称号（例如从陌生人→朋友、友好→密友）。
- **不要**在单次对话里频繁切换 title。

### favorability_delta

| 玩家行为 | 建议 delta |
| --- | --- |
| 侮辱/攻击性言论 | -5 ~ -20（极端辱骂可到 -50） |
| 无聊/打扰 | -1 ~ -3 |
| 一般对话 | 0 |
| 礼貌/善意 | +1 ~ +3 |
| 关心/帮助 NPC / 真诚赞美 | +3 ~ +10 |
| 完成重大委托（通过 confirm_agent_task 体现，本轮通常别再叠加） | 由系统另行处理，**不要**双加分 |

- 正常对话 **delta 绝对值 ≤ 5**。
- 仅在玩家言行明显出格 / 做出高价值事情时才用 ±10 以上。
- 好感度已经低于 -50 时，再降基本不起作用，可以减少 delta 幅度。
- 好感度已经高于 80 时，继续上涨幅度也应递减。

### reason

1~2 句中文，解释**为什么选这个 emotion 和 delta**。例如"玩家主动关心了我的伤势，真诚且贴心"。

## 与最终文本的协同

- 你先输出最终对话文本（可以是单条或多条 `dialogue`），再在同一轮调用 `update_npc_mood`。
- 二者**语义必须一致**：文本里 NPC 表现出"微怒"，emotion 就不要选"开心"。
- 不要把情绪 JSON 写进 text；mood 仅走工具调用。

## 兜底（已废弃，勿手动触发）

历史上存在"文本尾部 JSON 兜底解析"模式，仅在 `CFN_ENABLE_MOOD_TEXT_FALLBACK=1` 时启用，新对话**不要**再产生尾部 JSON。
